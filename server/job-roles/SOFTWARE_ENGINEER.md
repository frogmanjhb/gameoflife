# Software Engineer - Job Role Document

## Core Purpose

Build small, useful digital tools that support learning during CivicLab and broader class tasks.

## Tangible Responsibilities

### Daily (5–10 min)

- Check the Software Requests board (a list of problems learners want solved)
- Choose 1 task to work on or continue
- Test the app with 1–2 users and capture feedback

### Weekly (30–45 min total across the week)

- Bug hunt in CivicLab
- Deliver one working micro-app or feature improvement
- Publish it in the Town Hub as a "plugin" or tool link
- Run a 2–3 minute demo to the class
- Log:
  - what problem it solves
  - how to use it
  - what changed after feedback

## What Counts as a "Simple App"

Keep it to tools that take minutes to use, not days to build.

### Examples that fit CivicLab

- Budget calculator (rent + savings + spending)
- Loan repayment planner
- Land affordability checker
- Biome requirements checklist
- Tax calculator (gross vs net)
- Tender scoring form (Planner + Mayor use it)
- "Shop basket" cost calculator
- Random event generator (teacher-triggered)
- Savings goal tracker

### Examples that fit general learning

- Times tables trainer (adaptive)
- Spelling practice quiz
- Flashcards (topic-based)
- Quick comprehension quiz builder
- Study timer / Pomodoro

## Rules and Boundaries (to keep it manageable)

- Must be educationally useful
- Must be small (one screen, one purpose)
- Must be finished in one week
- No chat features, no accounts, no messaging
- Teacher approves requests before build starts

## Town Hub Permissions

- View Software Requests board
- Create and publish plugins/tools
- Access development tools and testing environment
- View usage statistics for published tools

## Job Details

- **Salary**: $6,000 per period
- **Bonus**: Additional pay if more than 5 students use the app created
- **Company**: Town Government / Tech Department
- **Location**: Development Lab
- **Category**: Technology & Innovation
